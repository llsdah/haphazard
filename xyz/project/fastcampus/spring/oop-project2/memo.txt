객체 지향
- 추상화 : 불필요 부분 제거 후 복잡성을 낮추는것
- 다형성 : 한개의 타입으로 여러 객체 참고
- 캡슐화
- 상송석

객체 5가지 설계원칙(SOLID),
- SRP : 단일 책임원칙
- OCP : 개발 폐쇠의 원칙
- LSP : 리스코프 치환의 원칙
- ISP : 인터페이스 분리의 원칙
- DIP : 의존성 역전의 원칙(변화의 영향을 받지 안도록)

객체 지향의 패러다임
- 적절한 객체에게 적절한 책임을 할당해 서로 ㅔ시지를 주고 받으려 혀벽하도록 하느 넋
- sw 복족도 낮추기 위한것
- 클래스 아닌 객체가 초점(역할과 책임)

high cohesion, loose coupling

객체 지향 설계시 고려 사항
- 도메인 구성 객체
- 객체 관계
- 동적 객체 정적 타입 추상화 도메인 모델링
- 협력 설계
- 객체 포관 타입 책임
